# Format: Markdown

## 타이포 위계

| 레벨 | 문법 | 용도 |
|------|------|------|
| L1 | `# 제목` (H1) | 문서 제목, 히어로 |
| L2 | `## 제목` (H2) | 섹션 제목 |
| L3 | `### 제목` (H3) | 소제목, 블록 제목 |
| L4 | 본문 | 설명, 상세 |
| XS | *이탤릭* 또는 작은 텍스트 | 캡션, 각주 |

## 여백 (빈 줄로 번역)

| 구간 | 빈 줄 수 |
|------|---------|
| 섹션(##) 간 | 3줄 |
| 블록 간 (같은 섹션 내) | 2줄 |
| 단락 간 | 1줄 |

## 강조 규칙

- **Bold**: 제목과 핵심 키워드에만. 본문에서 Bold 남발 금지 (1블록 1~2개 이내)
- *이탤릭*: 캡션, 출처, 부가정보에만
- ~~취소선~~: 사용 금지
- 밑줄: 사용 금지

## 3색 번역

- 콜아웃 사용시 전 문서에서 **1종만** (예: `> [!note]`만 쓰면 끝까지 note만)
- 색상·타입 난사 = FAIL

## 정렬

md에서는 전부 좌측정렬 (C5 중앙정렬 면제). 표 내부만 정렬 지정 가능.

## 구분

`---` (hr) = 최상위 섹션 구분에서만. 나머지는 빈 줄로.

## 하이라이트

- 옵시디언: `==핵심 구절==`
- GitHub/일반: `<mark>핵심 구절</mark>`

## 체크리스트

- [ ] H1~H3 + 본문 = 4단계 위계
- [ ] Bold 과용 없음 (1블록 2개 이내)
- [ ] 콜아웃 1종 통일
- [ ] 섹션 간 빈 줄 3개
